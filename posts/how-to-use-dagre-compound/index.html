使用 dagre-compound 库实现分组复合图布局

Jan 18 2021-01-18T00:00:00+08:00 by simplejason
13 min

本文所有代码均可在 此仓库 获取，或者在文中的在线示例直接体验。

dagre-compound 是什么？

dagre-compound: 基于 dagre 的多级复合图布局库，本文基于 0.0.5 版本实现

在很多业务场景下，我们都需要实现图可视化功能，以便给用户提供更好的交互式操作，但是对于多层复合图及分组图的实现，目前并没有比较好的解决方案，而本文介绍的布局库能够给我们提供良好的分组布局算法，利用计算结果可以很容易地和 d3 等渲染引擎结合。

应用场景

在日常业务中，对于分组复合图的需求经常出现，在一些渲染包含分组信息的复合图以及编辑复杂的数据结构情况下，手动更改 json / SQL 极其容易出现错误，用户体验也不是很好，可视化编辑功能可以很好地提升用户编辑体验，降低手动修改源数据出现错误的风险。

使用方式

dagre-compound 源码即将发布，对 G6 的支持也在计划内，目前我们已经可以通过 npm 下载来抢先体验。

创建一个项目空间，初始化 package.json 后执行

1

$ npm i dagre-compound dagre --save

因为其依赖 dagre 库，所以我们同样需要安装 dagre，至此，我们已经成功安装了 dagre-compound 布局库。

const data = {
    nodes: [
      { id: '0-1'},

const renderInfo: HierarchyGraphNodeInfo = buildGraph(data); // 使用 renderInfo 来渲染

UI 分析

你会看到我们的应用中包含四个部分。我已经将每个部分展示的区域标注了颜色:

  • 绿色：整个 graph 区域，内部包含节点与边
  • 红色：group 节点，包含子节点与边（子节点也可以为 group 节点）
  • 蓝色：基础节点，最小单位 node，无子节点
  • 黄色：边信息

可以看到，绿色部分与红色部分承担的功能都是一样的，都用于渲染子节点与边，于是我们进一步可将组件拆分为后三个部分。仅需要处理 group 节点、基础节点 node 与 边即可，内部如果有嵌套分组节点可复用 group 节点组件。 接下来本文会分别在 React 与 Angular 环境利用 dagre-compound 实现可嵌套分组布局，因为很多相似的设计理念，所以部分重复内容不会重复书写。 所以组件层级类似下述结构：

React 中实现分组布局

React 文档 ”Thinking in React“ 一节介绍了思考如何构建一个应用，我们同样采用 React 哲学的思维来处理。因为我们在之前已经分析过页面结构并拆分，所以我们直接从 UI 实现开始。

确定 UI 层级

我们首先创建 UI 组件，不涉及任何计算交互，方便我们清晰地查看拆分后的组件。因为前文我们已经知道有哪些组件需要拆分，所以我们通过 Class 来分别实现。

export class SvgContainer extends React.Component {}
// group node，处理组节点，root节点也可以理解为组节点
export class GroupNode extends React.Component {}
// edge 边组件，处理边渲染
export class GraphEdge extends React.Component {}
// node 组件，处理节点渲染，如果是展开的 group 节点，则复用 GroupNode 递归渲染
export class GraphNode extends React.Component {}

实现组件

既然需要实现的组件均已拆分完成，除了 SvgContainer 承担计算布局的工作，其他组件均只承担渲染工作，我们一起来逐步向每个组件中增加 render 函数。 对于边，我们使用 d3 line 方法来渲染

我们主要介绍如何复用 Group 组件，看一下 group 组件 和 node 组件的 render 函数（简洁版），我们通过复用 GroupNode 组件来渲染嵌套分组数据， 点击 在线示例 来看一下效果：

export class GroupNode extends React.Component<GroupNodeProps> {
  render() {
    const { type, renderInfo } = this.props;
    const edges = renderInfo.edges.map(edge => (
      <GraphEdge key={edge.v + "--" + edge.w} edge={edge} />
    ));
    const nodes = renderInfo.nodes.map(node => (
      <GraphNode key={node.name} node={node} />
    ));
    return (
      <g className={type}>
        <g className="core">
          <g className="edges">{edges}</g>
          <g className="nodes">{nodes}</g>
        </g>
      </g>
    );
  }
}
// Group Edge
export class GraphEdge extends React.Component<GraphEdgeProps> {
  render() {
    const { edge } = this.props;
    const points = line(edge.points)!;
    return (
      <g className="edge" key={edge.v + "--" + edge.w}>
        <path
          className="edge-line"
          data-edge={edge.v + "--" + edge.w}
          d={points}
        />
      </g>
    );
  }
}
// Node
export class GraphNode extends React.Component<GraphNodeProps> {
  render() {
    const { node } = this.props;
    return (
      <g key={node.name}>
        <g>
          <rect width={node.width} height={node.height} />
          <text>
            <tspan dy="1em" x="1">
              {node.name}
            </tspan>
          </text>
        </g>
		// 这里来判断，如果是展开的 group 节点，直接复用 GroupNode 实现递归渲染
        {(node as HierarchyGraphNodeInfo).expanded && (
          <GroupNode
            renderInfo={node as HierarchyGraphNodeInfo}
            type={"group"}
          />
        )}
      </g>
    );
  }
}

Angular 中实现分组布局

React 与 Angular 均是单向数据流，但是与 React 有所不同的是，Angular 支持双向绑定，所以我们在 ts 文件中修改的数据，HTML 模板会在脏值检测后更新。如果想了解更多，可以直接在 Angular 官网 查看，下面这部分的实现同样基于对 Angular 有所了解的情况。

点击 在线示例 查看渲染结果

实现组件

Angular 采用组件（Component）与 指令（Directive）来处理数据计算逻辑，通过 @Input 来传递数据，对应 React 的 Props，思想上是与 React.Component 类似。我们一样通过拆分组件的形式来实现渲染布局。

        </ng-container>
	</svg:g>
</ng-template>

边（Edge）与节点（Node）的渲染方式与 React 类似，dagre-compound 为我们提供了位置信息及宽高属性，我们用同样的方法实现它。（在线示例 详细地展示了代码是如何实现的）

总结

基于 React 与 Angular，我们通过 dagre-compound 计算出布局信息，然后借助 SVG 元素很轻易地实现了包含分组信息的 DAG 图可视化，我们还可以在 g 元素上通过 foreignObject 实现更复杂的 dom 结构（注意兼容性问题）。 因为底层计算库与渲染层拆分，允许我们轻易地自定义渲染层的交互而不必关心底层布局算法，这也意味着它拥有更强的渲染层定制化能力，感兴趣的同学可以基于在线示例添加更多动画、点击等事件，满足更多的业务应用场景。 下一章我们一起来探索如何去实现及优化复合分组图布局！
